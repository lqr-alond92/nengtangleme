# PRD 管理索引

本目录用于管理「能躺了吗」的产品需求文档。原则是：重要产品方案不覆盖旧版本，只新增版本。

## 版本规则

- `v0.x`：讨论期版本，产品方向、流程或信息架构仍在调整。
- `v1.0`：确认进入开发的正式版本。
- `v1.x`：开发中补充小调整，不改变核心方向。
- `v2.0`：产品定位、核心流程或信息架构发生重大变化。

## 状态定义

- `Draft`：正在讨论。
- `Review`：等待确认。
- `Approved`：已确认，可进入设计或开发。
- `Implementing`：正在开发。
- `Shipped`：已上线。
- `Superseded`：已被新版本替代，但保留历史。

## 使用流程

1. 每个独立产品方案建立一个 PRD 文件夹。
2. 每次核心方案变化时新增一个版本文件。
3. 小范围文字修正可以直接修改当前草案。
4. 方案确认后，复制为 `v1.0.md` 并标记 `Approved`。
5. 开发计划和代码实现只以 `Approved` 版本为准。
