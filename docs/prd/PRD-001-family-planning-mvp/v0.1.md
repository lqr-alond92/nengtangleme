# 能躺了吗 MVP 产品需求文档

版本：v0.1  
日期：2026-05-23  
阶段：MVP 产品设计

## 1. 产品定位

「能躺了吗」是一个面向家庭的长期财务规划与“躺平可行性”评估工具。

它不是记账工具，也不是复杂理财系统。第一版要解决的问题是：

- 用户是否知道自己家庭财务结构是否稳；
- 用户是否知道未来人生目标能否被当前资源支撑；
- 用户是否知道哪些目标必须保、哪些目标需要取舍；
- 用户是否能在产生大额消费、换工作、提前退休等想法时，快速判断它对家庭规划的影响。

MVP 的核心体验是：

> AI 通过聊天式问诊，引导用户一步步建立家庭财务地图，并生成一份可持续维护的规划页。

## 2. MVP 目标

### 2.1 用户目标

用户首次进入产品后，应能完成以下事情：

1. 通过 AI 破冰理解产品价值；
2. 在不面对复杂表单的情况下，逐步补全家庭财务核心信息；
3. 获得一份第一版家庭财务规划；
4. 明确看到家庭当前是否具备“躺”的条件；
5. 知道最关键的风险和下一步行动；
6. 后续可以回到 App 更新资产、目标或通过 AI 测试新决策。

### 2.2 产品目标

MVP 要验证：

1. 用户是否愿意在 AI 引导下填写家庭财务信息；
2. AI 总结是否能显著增强用户对产品的感知价值；
3. “资产 / 规划 / 目标”三页结构是否足够承载长期使用；
4. 7 个核心指标是否能作为稳定的理性证据层；
5. AI 常驻入口是否能承接用户后续的补充、追问和决策测试。

## 3. MVP 范围

### 3.1 本期必须做

1. 新人首次访问破冰页；
2. AI 聊天式问诊流程；
3. 家庭资源信息采集；
4. 必须目标与想要目标采集；
5. 默认参数确认；
6. 家庭规划计算；
7. 三层结果页；
8. 正式 App 框架：资产、规划、目标；
9. 规划页内长期 AI 入口；
10. 本地数据保存。

### 3.2 本期暂不做

1. 银行卡、证券账户、支付宝、微信等真实账户自动同步；
2. 多用户家庭协同；
3. 复杂税务规划；
4. 保险精算；
5. 投资组合推荐；
6. 真实支付；
7. 多端账号体系；
8. AI 自动执行金融操作。

## 4. 信息架构

MVP 采用 3 个底部 Tab：

1. 资产
2. 规划
3. 目标

不设置独立 AI Tab。AI 是规划页里的常驻能力。

### 4.1 资产页

用途：维护家庭当前财务底盘。

包含模块：

- 总资产；
- 总负债；
- 当前净资产；
- 可立即动用资产；
- 年度收入；
- 年度支出；
- 年度可规划结余。

### 4.2 规划页

用途：展示核心判断和 AI 解释，是 App 的主工作台。

包含模块：

1. 懂你的 AI；
2. 核心判断区；
3. 理性的我；
4. 理性行动的我；
5. 我的资产入口；
6. 我的目标入口；
7. 长期 AI 入口。

### 4.3 目标页

用途：维护未来人生目标。

目标分为两类：

- 必须要目标；
- 想要目标。

目标类型分为两类：

- 一次性支出；
- 持续性支出。

## 5. 新人首次访问流程

### 5.1 破冰首页

页面主标题：

> 你现在的财务焦虑，到底是不是多想？

副标题：

> 不是安慰你，也不是吓你。我们会用你的真实数据，帮你看清哪些担心只是情绪，哪些风险已经写在数字里。

AI 对话引导：

1. 你可能不是想躺平。
2. 你只是想知道，自己到底有没有退路。
3. 先选一个最担心的问题，我来帮你算清楚。

入口选项：

1. 生成我的家庭财务规划；
2. 目标都想要，最后谁先出局；
3. 这笔大钱，是消费还是透支。

MVP 中，第一项为主流程。第二、三项作为后续场景入口，MVP 可先引导回主流程。

## 6. AI 问诊流程

### 6.1 核心原则

AI 问诊必须遵守：

1. 以聊天消息形式呈现；
2. 每次只问一个问题；
3. 用户回答后，AI 先归纳，再进入下一题；
4. 不一次性抛出全部字段；
5. 用户点“自己输入”时必须出现真实输入框；
6. 用户点“帮我拆开算”时必须进入拆解模块；
7. 用户可以随时自由输入补充情况；
8. AI 必须先归类，再询问用户是否纳入计算。

### 6.2 问诊开始

用户点击：

> 生成我的家庭财务规划

系统生成用户消息：

> 我想生成一份家庭财务规划。

AI 回复：

> 我先帮你搭一张家庭财务地图。你不用一次填完，我会一个问题一个问题问，只收集会影响人生计划的数字。

### 6.3 问诊状态机

#### Step 1：总资产

AI 问：

> 你家现在总资产大概有多少？不需要精确，按万元填就行。

输入方式：

- 快捷选项；
- 数字输入；
- 不确定。

写入字段：

`resource.totalAssets`

如果用户选择“不确定”：

- 写入 `null`；
- 标记 `confidence.resource.totalAssets = low`；
- 后续报告中提示该字段会影响净资产率和资产负债率。

AI 归纳：

> 我先记为：当前总资产 X 万。

#### Step 2：总负债

AI 问：

> 这些资产里，负债大概还有多少？比如房贷、车贷、信用贷。

写入字段：

`resource.totalDebt`

计算：

`netWorth = totalAssets - totalDebt`

AI 归纳：

> 收到，当前净资产约 X 万。这个数字会作为家庭底盘进入测算。

#### Step 3：年度收入

AI 问：

> 你家一年税后大概能赚多少钱？包括工资、经营收入和稳定副业。

写入字段：

`cashflow.annualIncome`

AI 归纳：

> 我会把年度家庭收入先记为 X 万，并标注为当前收入支撑。

#### Step 4：年度可规划结余

AI 问：

> 一年下来，大概能剩下多少钱用于长期规划？如果不确定，我可以帮你拆开算。

选项：

- 直接输入年度结余；
- 帮我拆开算；
- 不确定。

如果选择“直接输入”：

写入：

`cashflow.annualSurplus`

如果选择“帮我拆开算”：

进入支出拆解模块，依次询问：

1. 每年基础生活支出；
2. 房贷或房租；
3. 孩子相关支出；
4. 父母赡养；
5. 保险；
6. 其他固定支出。

计算：

`annualSurplus = annualIncome - annualExpense`

AI 归纳：

> 先按年度可规划结余 X 万计算。后续可以再拆成稳定结余和不确定结余。

#### Step 5：必须目标多选

AI 问：

> 先看底线目标。未来哪些事情是家庭必须保障、不能轻易放弃的？可以一次选多个。

选项：

- 基本退休生活；
- 孩子教育；
- 父母养老；
- 医疗风险；
- 房贷 / 换房压力；
- 自己输入。

写入字段：

`goals.required[]`

选择多个目标后，AI 生成待补全队列。

AI 归纳：

> 我会先补全这些必须目标，再看它们能不能被当前资源覆盖。

#### Step 6：逐个补全必须目标

对每个必须目标，AI 依次询问：

1. 这是一次性支出，还是持续性支出？
2. 大概几年后开始需要？
3. 如果是一次性，需要多少钱？
4. 如果是持续性，每月或每年需要多少？
5. 持续到什么时候？
6. 有没有最低可接受版本？

写入字段：

`goals.required[n]`

字段结构见第 7 章。

AI 归纳示例：

> 已记录：孩子教育，12 年后开始，一次性预算 80 万，属于必须目标。

#### Step 7：想要目标多选

AI 问：

> 底线目标记录好了。接下来看看想要但可以取舍的目标，可以多选，也可以跳过。

选项：

- 提前退休；
- 长期旅行；
- 换更大的房子；
- 买车；
- 创业 / 自由职业；
- 暂时没有；
- 自己输入。

写入字段：

`goals.optional[]`

想要目标额外询问：

> 如果资源不够，这个目标可以延期、降配，还是宁愿保留？

写入：

`goal.flexibility`

#### Step 8：默认假设确认

AI 问：

> 我先按保守默认值计算：预期收益率 4%，通胀率和折现率 3%。先按这个算可以吗？

选项：

- 按默认先算；
- 我要修改假设。

默认写入：

```js
assumptions.expectedReturnRate = 0.04
assumptions.inflationRate = 0.03
assumptions.discountRate = 0.03
```

## 7. 数据模型

### 7.1 Resource

```js
resource = {
  totalAssets: number | null,
  totalDebt: number | null,
  liquidAssets: number | null,
  netWorth: number | null
}
```

### 7.2 Cashflow

```js
cashflow = {
  annualIncome: number | null,
  annualExpense: number | null,
  annualSurplus: number | null,
  stableAnnualSurplus: number | null
}
```

### 7.3 Goal

```js
goal = {
  id: string,
  name: string,
  category: "required" | "optional",
  type: "one_time" | "recurring",
  priority: number,
  amount: number,
  frequency: "once" | "monthly" | "yearly",
  startYear: number,
  endYear: number | null,
  flexibility: "fixed" | "delayable" | "reducible" | "optional",
  minimumAcceptableAmount: number | null
}
```

### 7.4 Assumptions

```js
assumptions = {
  expectedReturnRate: number,
  inflationRate: number,
  discountRate: number,
  includeFutureIncome: boolean,
  workingYears: number | null
}
```

### 7.5 AI Context

```js
aiContext = {
  messages: Message[],
  extractedFacts: ExtractedFact[],
  pendingConfirmations: PendingConfirmation[],
  confidence: Record<string, "high" | "medium" | "low">
}
```

## 8. 计算规则

### 8.1 当前净资产

```txt
当前净资产 = 总资产 - 总负债
```

### 8.2 年度可规划结余

```txt
年度可规划结余 = 年度收入 - 年度支出
```

如果用户直接输入年度结余，则以用户输入为准。

### 8.3 未来收入现值

```txt
未来收入现值 = Σ 第 t 年年度可规划结余 / (1 + 折现率)^t
```

计算范围：

`t = 1` 到 `workingYears`

### 8.4 一次性目标现值

```txt
一次性目标现值 = 目标金额 / (1 + 折现率)^距今年数
```

### 8.5 持续性目标现值

```txt
持续性目标现值 = Σ 第 t 年目标支出 / (1 + 折现率)^t
```

如果用户按月填写：

```txt
年度目标支出 = 月度支出 * 12
```

### 8.6 可规划资源

```txt
可规划资源 = 当前净资产 + 未来收入现值
```

### 8.7 目标覆盖率

```txt
目标覆盖率 = 可规划资源 / 全部目标现值
```

### 8.8 覆盖后余量

```txt
覆盖后余量 = 可规划资源 - 全部目标现值
```

## 9. 理性的我：7 个核心指标

`理性的我` 必须展示完整 7 个指标，采用横向卡片。

每张卡片包含：

1. 指标名称；
2. 当前数值；
3. 状态：达标 / 预警 / 危险；
4. 一句白话解释；
5. 查看精算过程：公式、代入数据、门槛。

### 9.1 净资产率

```txt
净资产率 = 当前净资产 / 总资产
```

门槛：

- 达标：>= 50%
- 预警：20% - 50%
- 危险：< 20%

### 9.2 资产负债率

```txt
资产负债率 = 总负债 / 总资产
```

门槛：

- 达标：<= 40%
- 预警：40% - 60%
- 危险：> 60%

### 9.3 年度结余率

```txt
年度结余率 = 年度可规划结余 / 年度收入
```

门槛：

- 达标：>= 30%
- 预警：10% - 30%
- 危险：< 10%

### 9.4 支出收入比

```txt
支出收入比 = 年度支出 / 年度收入
```

门槛：

- 达标：<= 70%
- 预警：70% - 90%
- 危险：> 90%

### 9.5 目标覆盖率

```txt
目标覆盖率 = 可规划资源 / 目标现值
```

门槛：

- 达标：>= 100%
- 预警：70% - 100%
- 危险：< 70%

### 9.6 现有净资产覆盖率

```txt
现有净资产覆盖率 = 当前净资产 / 目标现值
```

门槛：

- 达标：>= 100%
- 预警：40% - 100%
- 危险：< 40%

### 9.7 未来收入依赖度

```txt
未来收入依赖度 = 未来收入现值 / 可规划资源
```

门槛：

- 达标：<= 40%
- 预警：40% - 70%
- 危险：> 70%

## 10. 结果页

新人问诊完成后，进入正式规划页，而不是停留在一次性报告页。

### 10.1 过渡态

AI loading 文案：

1. 正在生成你的家庭财务地图；
2. 正在折现未来目标；
3. 正在检查 7 个关键指标；
4. 正在判断哪些目标最挤压计划；
5. 正在把对话整理成你的规划页。

过渡完成后，清空聊天流上下文的视觉内容，进入规划页顶部。

### 10.2 懂你的 AI

作用：用锋利但不恐吓的语言给出结论。

示例：

> 你现在不是不能躺，而是必须先保住必要目标，再判断想要目标能不能上车。

### 10.3 核心判断区

展示：

- 当前答案；
- 资源覆盖目标比例；
- 覆盖后余量；
- 必须目标覆盖情况；
- 想要目标取舍压力。

### 10.4 理性的我

展示 7 个核心指标。

### 10.5 理性行动的我

给 3 条行动建议。

要求：

- 必须具体；
- 不说“少花钱”“多存钱”；
- 必须能在本周开始执行。

### 10.6 我的资产 / 我的目标

规划页下方展示两个入口：

- 我的资产；
- 我的目标。

移动端可上下或双卡展示。

## 11. 长期 AI 入口

规划页底部、导航栏上方保留 AI 入口。

入口文案：

> 继续问 AI：补充情况、测试决定、调整目标

用户可输入：

- 我想买车，会不会影响退休目标？
- 我还有一笔理财半年后到期；
- 我想把孩子教育目标提高到 150 万；
- 如果我 45 岁换轻松工作呢？

AI 必须先判断用户意图类型：

- 修改资产；
- 修改负债；
- 修改收入；
- 修改支出；
- 新增目标；
- 修改目标；
- 新增大额消费决策；
- 修改假设参数。

AI 归类后必须询问用户确认，再更新结构化数据并重新计算规划页。

## 14. AI 接入原则

MVP 上线版需要接入真实大模型。

原则：

1. 前端不暴露模型 API Key；
2. 通过后端接口调用模型；
3. AI 输入为结构化财务数据 + 最近上下文；
4. AI 输出分为两部分：
   - 用户可读总结；
   - 结构化更新建议；
5. AI 不直接覆盖用户数据，必须让用户确认。

## 15. 成功标准

MVP 成功标准：

1. 用户能完成首次问诊；
2. 用户能理解结果页的核心结论；
3. 用户能看到 7 个指标证据；
4. 用户能维护资产和目标；
5. 用户能通过 AI 入口补充情况；
6. 用户能在大额决策前回到 App 测算影响。
