## 0. v1.2 相对 v1.1 的调整摘要

原版本：`PRD-001 v1.1`  
调整版本：`PRD-001 v1.2`

本次调整不改变产品定位、三 Tab 信息架构、新人破冰节奏、必须/想要目标结构和真实大模型接入方向。它主要修正家庭资产规划中的一个关键误区：

> 账面资产不等于可直接用于人生目标的钱。

### 0.1 资产口径升级

v1.1：

- 资源侧主要使用总资产、总负债、当前净资产、年度收入、年度支出。
- 计算容易把所有净资产都视为可规划资源。

v1.2：

- 资产拆成两个核心口径：
  - 可立即动用资产；
  - 不易变现资产。
- 总资产由两者推算：

```text
总资产 = 可立即动用资产 + 不易变现资产
当前净资产 = 总资产 - 总负债
```

### 0.2 规划资源拆成两套视角

v1.1：

```text
可规划资源 = 当前净资产 + 未来结余现值
```

v1.2：

```text
账面资源 = 当前净资产 + 未来结余现值
可调用资源 = 可立即动用资产 + 未来结余现值
```

页面必须同时呈现两套视角，让用户看到：

- 账面上是否看起来够；
- 真正能调用的钱是否够。

### 0.3 AI 总结增加认知冲击

v1.1：

- AI 总结主要围绕目标覆盖率、负债率、结余率、未来收入依赖度。

v1.2：

- 当账面覆盖率明显高于可调用覆盖率时，AI 必须指出资产锁定风险。
- 首屏结论优先表达：

> 你不是没资产，而是大部分资产不能直接替目标付款。

### 0.4 理性指标层级调整

v1.1：

- `理性的我` 展示 7 个核心指标，容易在首屏占据过多注意力。

v1.2：

- 首屏先展示资产可调用性反差。
- `理性的我` 中 7 个指标仍保留，但新增资产结构相关指标：
  - 可动用资产占比；
  - 不易变现资产占比；
  - 可调用覆盖率。
- 指标文案最多两行，完整公式放在展开态。

## 1. 产品定位

「能躺了吗」是一个面向家庭的长期财务规划与“躺平可行性”评估工具。

MVP 的核心体验是：

> AI 通过聊天式问诊，引导用户一步步建立家庭财务地图，并生成一份可持续维护的规划页。

v1.2 强化一个产品差异点：

> 很多家庭不是没有资产，而是资产结构不灵活。产品必须帮助用户看清“账面有钱”和“真正能调度的钱”之间的差距。

## 2. MVP 目标

### 2.1 用户目标

用户首次进入产品后，应能完成以下事情：

1. 通过 AI 破冰理解产品价值；
2. 在不面对复杂表单的情况下，逐步补全家庭财务核心信息；
3. 区分可立即动用资产和不易变现资产；
4. 获得一份第一版家庭财务规划；
5. 明确看到家庭当前是否具备“躺”的条件；
6. 知道自己账面资源和可调用资源是否存在明显断层；
7. 后续可以回到 App 更新资产、目标或通过 AI 测试新决策。

### 2.2 产品目标

MVP 要验证：

1. 用户是否愿意在 AI 引导下填写家庭财务信息；
2. AI 总结是否能显著增强用户对产品的感知价值；
3. “资产 / 规划 / 目标”三页结构是否足够承载长期使用；
4. 资产可调用性是否能成为用户认知冲击点；
5. AI 常驻入口是否能承接用户后续的补充、追问和决策测试。

## 3. 信息架构

MVP 采用 3 个底部 Tab：

1. 资产
2. 规划
3. 目标

不设置独立 AI Tab。AI 是规划页里的常驻能力。

### 3.1 资产页

用途：维护家庭当前财务底盘。

包含模块：

- 可立即动用资产；
- 不易变现资产；
- 总资产；
- 总负债；
- 当前净资产；
- 年度收入；
- 年度支出；
- 年度可规划结余。

### 3.2 规划页

用途：展示核心判断和 AI 解释，是 App 的主工作台。

包含模块：

1. 懂你的 AI；
2. 账面资源与可调用资源对比；
3. 理性的我；
4. 行动的我；
5. 我的资产入口；
6. 我的目标入口；
7. 长期 AI 入口。

### 3.3 目标页

用途：维护未来人生目标。

目标分为两类：

- 必须要目标；
- 想要目标。

目标类型分为两类：

- 一次性支出；
- 持续性支出。

## 4. 新人问诊流程调整

### 4.1 资产问诊顺序

v1.2 中，资源问诊从“总资产视角”改成“能否动用视角”：

1. 可立即动用资产；
2. 不易变现资产；
3. 总负债；
4. 年度收入；
5. 年度可规划结余或年度支出；
6. 必须目标；
7. 想要目标；
8. 默认假设确认。

### 4.2 关键问诊文案

可立即动用资产：

> 先不看房子这些大件。现在家里真正能拿出来用的钱，大概有多少？

不易变现资产：

> 再看那些账面上有、但不太能直接花的钱，比如房子、车、长期资产，大概有多少？

总负债：

> 这些资产背后，还有多少负债？比如房贷、车贷、信用贷。

### 4.3 输入组件要求

每个输入组件必须包含：

- 当前字段标题；
- 简短解释；
- 单位；
- 当前问题专属快捷值；
- 自定义输入；
- 发送按钮。

金额单位统一为万元。

## 5. 数据模型

### 5.1 资源字段

```ts
type Resource = {
  liquidAssets: number | null,
  lockedAssets: number | null,
  totalAssets: number | null,
  liabilities: number | null,
  netWorth: number | null,
  annualIncome: number | null,
  annualExpense: number | null
}
```

MVP 前端实现中可保持字段名：

```ts
type Plan = {
  liquidAssets: number,
  lockedAssets: number,
  assets: number,
  liabilities: number,
  annualIncome: number,
  annualExpense: number,
  goals: Goal[]
}
```

其中：

```text
assets = liquidAssets + lockedAssets
```

旧数据兼容：

- 如果旧数据只有 `assets`，没有 `liquidAssets` 和 `lockedAssets`：
  - 不崩溃；
  - 默认 `liquidAssets = 0`；
  - 默认 `lockedAssets = assets`；
  - AI 提示可立即动用资产需要复核。

## 6. 核心计算

### 6.1 基础计算

```text
总资产 = 可立即动用资产 + 不易变现资产
当前净资产 = 总资产 - 总负债
年度结余 = 年收入 - 年支出
未来结余现值 = 每年结余按折现率折回今天
目标需求现值 = 所有未来目标按折现率折回今天
```

### 6.2 两套资源口径

```text
账面资源 = 当前净资产 + 未来结余现值
可调用资源 = 可立即动用资产 + 未来结余现值
```

### 6.3 两套覆盖率

```text
账面覆盖率 = 账面资源 / 目标需求现值
可调用覆盖率 = 可调用资源 / 目标需求现值
```

### 6.4 资产结构指标

```text
可动用资产占比 = 可立即动用资产 / 总资产
不易变现资产占比 = 不易变现资产 / 总资产
```

## 7. 结果页叙事

### 7.1 懂你的 AI

首屏必须先定调，不先抛公式。

优先使用以下逻辑：

1. 如果账面覆盖率高，但可调用覆盖率低：

> 你不是没资产，而是大部分资产不能直接替目标付款。

2. 如果账面覆盖率和可调用覆盖率都低：

> 现在不是少一个目标的问题，而是可调用资源和未来目标之间有硬缺口。

3. 如果两者都充足：

> 你的资源不仅账面够，真正能调用的钱也有余量。

### 7.2 关键反差卡

必须展示：

- 可调用资源；
- 账面资源；
- 目标需求；
- 可调用覆盖率；
- 账面覆盖率。

### 7.3 理性的我

保留完整指标，但首屏弱化。

v1.2 指标至少包含：

1. 可调用覆盖率；
2. 账面覆盖率；
3. 可动用资产占比；
4. 不易变现资产占比；
5. 净资产率；
6. 资产负债率；
7. 年度结余率；
8. 未来收入依赖度。

完整公式放在“查看精算过程”展开态。

### 7.4 行动的我

行动建议必须围绕资产结构展开，不给空泛建议。

示例：

1. 把资产拆成“能 30 天内动用”和“不能轻易动”的两张清单。
2. 对最大必须目标做一版“不卖长期资产”的测算。
3. 如果不易变现资产占比过高，先给未来 12 个月做现金流安全表。

## 8. 验收标准

### 8.1 数据与计算

- 用户只填可立即动用资产和不易变现资产时，总资产能正确推算。
- 旧数据只有 `assets` 时不崩。
- 可调用资源和账面资源分别计算。
- 可调用覆盖率和账面覆盖率分别计算。

### 8.2 AI 总结

- `lockedAssets` 高、`liquidAssets` 低时，AI 必须指出资产锁定风险。
- 账面覆盖率明显高于可调用覆盖率时，首屏必须出现认知反差。
- 总结不应变成长报告。

### 8.3 页面体验

- 资产页能清楚区分可立即动用资产与不易变现资产。
- 规划页首屏能看到两套资源口径。
- 7 个以上指标仍可查看，但不抢主角。
- 移动端不溢出、不遮挡、不乱码。
