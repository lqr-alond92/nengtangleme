# 能躺了吗 MVP 产品需求文档

版本：v1.4  
日期：2026-06-05  
状态：Approved  
阶段：Cleo 对照方案与核心判断六段式规划页

## 0. v1.4 相对 v1.3 的调整摘要

原版本：`PRD-001 v1.3`  
调整版本：`PRD-001 v1.4`  
讨论原型：`docs/ai-judgement-chat-demo.html`

本次调整不改变三 Tab 信息架构、资产口径、目标现值计算、补充规划对话流。它把规划页从“指标展示页”进一步升级为“AI 先给判断，再落到计划”的叙事型工作台。

核心变化：

1. 以 Cleo 类产品为对照，保留对话亲近感，但不做泛聊天；
2. 规划页第一屏先回答用户真正关心的问题：现在能不能躺，为什么；
3. 页面采用六段式结构：一句话承接、核心结论、核心指标支撑、压力来源、长期建议、短期计划；
4. 长期建议必须指出一个最该调整的目标，并给出调整方向；
5. 短期计划必须落成收入、支出、结余和复盘节奏四个数字化设置。

## 1. 背景

当前规划页已经能展示覆盖率、风险、行动项和长期维护入口，但用户第一次看到结果时，仍需要自己把多个指标串起来理解。

与 Cleo 的对照给出的启发是：用户不是先要完整报表，而是先要一个能听懂人话的判断。Cleo 的强项是对话亲近感和快速反馈；「能躺了吗」不应该复制消费记账式聊天，而应把这种亲近感用于家庭长期规划：先把结论讲清楚，再用指标证明，再给可执行的长期和短期计划。

## 2. 产品目标

- 让用户进入规划页后，3 秒内知道系统判断是什么；
- 把“能不能躺”的回答从指标堆叠变成清晰叙事；
- 保留财务规划的可信度，关键判断必须有指标支撑；
- 把长期目标调整和本周执行计划放在同一条决策链里；
- 让本版本可以直接迁移进现有 React App，而不是停留在 HTML 原型。

## 3. 用户与场景

目标用户：

- 正在判断家庭是否可以降低工作强度、提前退休或进入半躺状态的人；
- 有换房、孩子教育、父母养老、退休生活等长期目标的人；
- 不想看复杂财务表，但需要知道“现在最该改什么”的用户。

核心场景：

1. 用户完成首次问诊，进入规划页；
2. 用户看到 AI 先复述自己的问题；
3. 系统给出一句核心判断；
4. 用户查看指标证据和压力来源；
5. 用户看到长期目标调整建议；
6. 用户确认本周执行计划或继续进入 AI 监测补充信息。

## 4. 范围

### 4.1 本期包含

- 在规划页沉淀六段式结果结构；
- 复用现有 `buildAiReport(plan, metrics)` 和 `buildMetrics(plan)` 的指标能力；
- 新增面向页面的判断摘要、压力来源、长期建议、短期计划；
- 让长期建议优先围绕最大压力目标生成；
- 让短期计划以当前年收入、年支出和年结余生成月度目标；
- 保留“重新问诊”“AI 监测”“资产/目标入口”。

### 4.2 本期不包含

- 不接入真实 Cleo 或外部产品 API；
- 不做自然语言完全自由聊天；
- 不改底层现值、覆盖率、资产拆分公式；
- 不新增后端；
- 不做账户同步或真实记账；
- 不把 HTML 原型作为正式页面直接嵌入。

## 5. 核心流程

1. 用户完成问诊或打开已有规划；
2. 规划页顶部出现“知心姐姐 / 家庭财务 AI”判断区；
3. AI 先用一句话承接用户意图；
4. AI 给出核心结论，说明是否可以躺、半躺或暂缓；
5. 页面展示四个关键指标：可调用覆盖、账面覆盖、目标缺口、年度结余率；
6. 页面展示当前主要压力来源；
7. 页面给出长期建议，默认围绕最大压力目标；
8. 页面给出今年短期计划：月收入目标、月支出上限、月结余目标、复盘节奏；
9. 用户可进入资产页、目标页、重新问诊或 AI 监测补充规划。

## 6. 页面与交互

### 6.1 一句话承接

位置：规划页顶部 AI 判断区。

文案目标：

- 复述用户最关心的问题；
- 不使用复杂公式；
- 建立“AI 听懂了”的感觉。

示例：

```text
你问的不是账上有多少钱，而是这个家庭能不能把工作强度慢慢降下来。
```

### 6.2 核心结论

结论必须直接回答“能不能躺”。

规则：

- 可调用覆盖率 >= 100%：可以推进，但仍需复核资产流动性；
- 可调用覆盖率 70%-100%：可以朝半躺推进，但不建议立刻停下长期收入；
- 可调用覆盖率 < 70%：暂时不能直接躺，先处理缺口或目标节奏。

### 6.3 核心指标支撑

展示四个指标：

- 可调用覆盖；
- 账面覆盖；
- 目标缺口；
- 年度结余率。

指标必须用短句解释，不只展示百分比。

### 6.4 压力来源

基于 `aiReport.risks` 和 `metrics.maxGoal` 展示主要压力。

优先说明：

- 是资产锁定；
- 是目标资金缺口；
- 是单点收入依赖；
- 是现金流脆弱；
- 还是最大目标挤压。

### 6.5 长期建议

长期建议必须包含：

- 建议调整哪个目标；
- 为什么先动它；
- 建议的第一版调整方式。

默认规则：

- 若存在最大目标，建议先对最大目标做“延后 3 年 + 降低 20% 预算”的保守版；
- 若不存在目标，建议先添加必要目标和想要目标；
- 若覆盖率已经达标，建议保留目标，但复核可调用资产。

### 6.6 短期计划

短期计划必须落成四项：

- 月收入目标；
- 月支出上限；
- 月结余目标；
- 复盘节奏。

默认计算：

- 月收入目标 = 年收入 / 12；
- 月支出上限 = 年支出 / 12；
- 月结余目标 = max(年收入 - 年支出, 0) / 12；
- 复盘节奏 = 每周记录一次，每月底复盘一次长期进度。

## 7. 数据与计算

本期不新增持久化字段，只新增页面展示层派生数据。

输入：

```text
plan
metrics
aiReport
```

派生展示数据：

```text
judgement.leadIn
judgement.verdict
judgement.reason
judgement.keyMetrics[]
judgement.pressure
judgement.longTermAdvice
judgement.shortTermPlan[]
```

所有派生数据必须可由现有计划数据重新生成，不写入 localStorage。

## 8. AI 能力

本版本的 AI 是产品表达层，不接入真实大模型。它使用规则化模板模拟“知心姐姐 / 家庭财务 AI”的输出。

输出原则：

- 先说判断，再说理由；
- 用用户能听懂的话解释指标；
- 不夸大确定性；
- 不给投资建议；
- 不替用户做不可逆决定；
- 涉及长期目标调整时，必须保留“用户可继续调整”的入口。

## 9. 验收标准

- 规划页顶部能看到一句话承接和核心结论；
- 页面能展示四个核心指标；
- 页面能展示压力来源；
- 页面能展示长期建议，并明确目标调整方向；
- 页面能展示短期计划四项数字；
- 资产、目标、AI 监测入口仍可用；
- 现有财务计算测试通过；
- 页面在移动端不出现文字重叠；
- `docs/ai-judgement-chat-demo.html` 作为讨论原型保留，正式实现进入 React App。

## 10. 版本记录

v1.4 是基于 Cleo 对照讨论沉淀出的规划页叙事升级版本。它将 `知心姐姐核心判断 Demo` 的六段式结构迁移进正式 App，并作为后续真实 AI 输出设计的基准。
