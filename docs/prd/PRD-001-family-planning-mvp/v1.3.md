# 能躺了吗 MVP 产品需求文档

版本：v1.3  
日期：2026-05-25  
状态：Approved  
阶段：AI 持续监测入口与补充规划对话流

## 0. v1.3 相对 v1.2 的调整摘要

原版本：`PRD-001 v1.2`  
调整版本：`PRD-001 v1.3`

本次调整不改变资产可调用口径、三 Tab 信息架构和结果页核心计算。它主要优化规划页的长期 AI 能力：

1. 右下角“继续问 AI”入口改为更符合“持续监测”的视觉表达；
2. 用户点击后不再停留在按钮层，而是回到 AI 对话流；
3. 用户可补充资产或新增目标；
4. 补充完成后系统重新生成规划页。

## 1. CEO Brief

用户在完成首次规划后，不应该只能看静态结果。规划页需要一个长期入口，让用户在发生新想法、新目标、新资产变化时，可以回到 AI 对话，补充信息，并得到新的规划结果。

## 2. PM 拆解

### 2.1 本次范围

- 修改悬浮 AI 入口的视觉与文案；
- 点击入口后进入补充规划对话；
- 支持两类 MVP 补充：
  - 更新资产结构；
  - 新增目标；
- 补充完成后进入 loading，再回到规划页；
- 规划页基于新数据重新计算。

### 2.2 不做范围

- 不接入真实大模型；
- 不做自然语言完全自动拆解；
- 不做多轮复杂自由对话；
- 不做账户同步；
- 不做后台实时监控。

## 3. UX Review

### 3.1 悬浮入口

文案从“继续问”调整为：

```text
AI 监测
```

视觉方向：

- 使用雷达 / 监测感图标；
- 增加呼吸光点或扫描环；
- 表达“AI 在持续观察规划变化”，而不是普通聊天入口。

### 3.2 点击后的对话流

入口点击后进入全屏对话：

```text
我会基于你现在的资产和目标继续补充，不会让你重填全部信息。
```

用户先选择补充类型：

1. 更新资产结构；
2. 新增一个目标。

MVP 先不做“任意自然语言完全解析”，但允许用户通过对话式组件逐步补全。

### 3.3 完成态

补充完成后进入 loading：

```text
正在重新计算你的家庭财务地图
```

随后回到规划页，并展示新结果。

## 4. Engineering Plan

### 4.1 数据逻辑

新增补充规划纯函数：

```text
applyPlanSupplement(plan, supplement)
buildSupplementGoal(input)
```

支持：

- 更新 `liquidAssets`、`lockedAssets`、`liabilities`；
- 新增一次性或持续性目标。

### 4.2 前端状态

App 新增补充模式：

```text
isSupplementing
```

当用户点击规划页卡片或悬浮入口：

1. `isSupplementing = true`
2. 展示 `SupplementExperience`
3. 完成后更新 plan
4. loading
5. 回到规划页

## 5. QA Plan

必须验收：

- 点击右下角 AI 入口能进入对话；
- 点击规划页“继续问 AI”卡片也能进入同一对话；
- 用户可更新资产结构；
- 用户可新增目标；
- 完成后出现 loading；
- loading 后回到规划页；
- 规划页基于新数据重新计算；
- 入口图标具备持续监测感，不再像普通问答按钮。
