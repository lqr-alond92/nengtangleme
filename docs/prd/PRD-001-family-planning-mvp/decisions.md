# PRD-001 关键决策记录

## 2026-05-23

### 不做复杂记账

产品不是每日记账工具，而是家庭长期财务规划与躺平可行性评估工具。

### AI 不做独立 Tab

AI 是规划页里的长期能力，不作为底部独立入口。底部导航固定为资产、规划、目标。

### 新人破冰必须先建立焦虑和退路感

首次访问先用偏动画和对话的形式建立氛围，再让用户选择最担心的问题。第一项进入主流程：生成我的家庭财务规划。

### 问诊必须一题一问

不能一次性抛出总资产、总负债、收入、支出等字段。AI 必须以消息形式逐步提问，用户答完一道题后再进入下一题。

### 目标分为必须要和想要

必须目标可以多选，并逐个补全。必须目标问完后，再询问想要目标。资源不足时，想要目标优先进入取舍逻辑。

### 上线版必须接入真实大模型

MVP 上线版需要接入真实模型。面向中国用户时优先考虑国内可访问模型，通过后端调用，前端不暴露 API Key。

## 2026-05-24

### 首页内容必须分阶段出现

三句 AI 对话和三个问题入口不能同时出现。先建立情绪和上下文，再给用户选择入口。

### 持续目标采用组合组件

退休生活等持续目标不再拆成多轮单题。用“类型/频率/金额”和“开始/结束/补充说明”两个组件完成，降低填写疲劳。

### 结果页采用四段叙事

结果页按“懂你的 AI -> 理性的我 -> 行动的我 -> 回到长期规划”组织。先定调，再证明，再行动，最后沉淀到长期使用。

### 资产拆成可立即动用和不易变现

家庭资产规划不能默认把所有资产都视为可规划资源。v1.2 起采用两个资产口径：可立即动用资产、不易变现资产。产品需要让用户看到“账面有资产”和“真正能调用的钱”之间的差距。

### 结果页必须呈现账面资源和可调用资源

规划结论不能只使用当前净资产口径。v1.2 起同时计算账面资源和可调用资源：账面资源用于理解家庭整体家底，可调用资源用于判断目标短中期支撑能力。

## 2026-05-25

### AI 入口表达持续监测而不是普通聊天

规划页右下角入口从“继续问”调整为“AI 监测”。视觉上采用雷达、扫描、呼吸点等语言，表达 AI 正在持续观察家庭财务地图。

### 点击 AI 入口后进入补充规划对话

AI 不作为独立 Tab，也不只是一个按钮。用户点击入口后进入对话流，基于当前规划补充资产或目标，完成后重新生成规划页。
