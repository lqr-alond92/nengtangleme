# 版本回滚与腾讯云上传记录

## 为什么不要只依赖压缩包

腾讯云 CloudBase 静态托管上传的是构建后的压缩包。压缩包适合发布，但不适合长期管理版本，因为它不方便看差异，也不方便恢复到某一次需求讨论后的源码状态。

后续推荐用 GitHub 管理源码版本，用本地 zip 包负责上传腾讯云：

- GitHub：保存源码、PRD、提示词、Demo 和每次可回滚的提交。
- 本地 zip 包：作为当次腾讯云上传物。
- 腾讯云：只负责运行当前发布版本。

## 当前建议流程

1. 完成一轮功能或视觉调整。
2. 本地验证：`npm test`、`npm run build`。
3. 生成 CloudBase 上传包。
4. 提交到 GitHub，并记录提交号。
5. 上传 zip 到腾讯云 CloudBase。
6. 线上验证通过后，在本文件补充记录。

## 回滚方式

如果腾讯云上线后发现问题：

1. 在 GitHub 找到上一个稳定提交。
2. 回到该提交对应源码。
3. 重新执行 `npm run build`。
4. 重新打包 `dist`。
5. 在腾讯云 CloudBase 静态网站托管里“更新服务”，上传旧版本包。
