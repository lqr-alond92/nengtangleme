# 手机 Alpha PWA 体验说明

本阶段目标是让体验用户直接在自己的手机上使用「能躺了吗」，不登录、不注册、不做云端账号保存。应用通过 HTTPS 静态地址分发前端文件；用户规划数据保存在当前手机浏览器里。AI 问答由 CloudBase 云函数代理 DeepSeek，体验用户不需要填写 API Key。

## 为什么不是 `127.0.0.1`

`127.0.0.1` 永远指向“当前设备自己”：

- 在开发者电脑上打开 `http://127.0.0.1:5173`，访问的是开发者电脑。
- 在用户手机上打开 `http://127.0.0.1:5173`，访问的是用户手机自己。

手机体验用户不能直接访问开发者电脑上的 localhost。真正可用的手机 Alpha 需要一个手机能打开的 HTTPS PWA 地址。

## 推荐交付形态

```text
开发者构建 dist/
        |
        v
HTTPS 静态托管地址
        |
        v
体验用户手机浏览器打开并添加到主屏幕
        |
        v
规划数据保存在手机浏览器 localStorage
```

静态托管只负责分发 HTML、CSS、JS、图标和 manifest，不保存用户的计划数据。DeepSeek API Key 只配置在 CloudBase 云函数环境变量中。

## 开发者部署步骤

1. 构建前端：

```bash
npm install
npm run build
```

2. 将 `dist/` 目录部署到 HTTPS 静态托管服务。

可选托管方式：

- GitHub Pages
- Cloudflare Pages
- Netlify
- Vercel
- 对象存储静态网站
- 任意能提供 HTTPS 的静态文件服务

3. 把部署后的 HTTPS 地址发给体验用户。

如果使用国内自有域名、国内云厂商或正式生产域名，按对应平台和地区要求处理备案、域名解析和 HTTPS。备案审批期间，可以先使用托管平台提供的临时 HTTPS 域名做小范围体验。

## 体验用户操作

1. 用手机浏览器打开体验地址。
2. 在浏览器菜单中选择“添加到主屏幕”。
3. 从手机桌面打开「能躺了吗」。
4. 完成首次问诊。
5. 进入“AI”页，点击“测试 AI 服务”。
7. 回到“规划”页继续问 AI。

## 数据与 Key 存储边界

当前实现使用浏览器 `localStorage`：

- `neng_tang_plan_v1`：用户规划数据。
- `neng_tang_onboarding_complete_v1`：是否完成首次问诊。
- `neng_tang_model_settings_v1`：旧版本地模型设置兼容项；当前托管 AI 版本不要求用户填写。

这些数据只存在当前浏览器环境里。用户换手机、换浏览器、清理站点数据或卸载 PWA 后，数据可能丢失。本阶段不提供账号同步和云端恢复。

## 模型调用边界

当前模型调用走 CloudBase 代理：

```text
用户手机浏览器 -> CloudBase 云函数 -> DeepSeek API
```

这避免了浏览器 CORS 限制，也避免体验用户手动填写 API Key。需要注意：用户提出的问题和当前规划摘要会经过 CloudBase 云函数，再转发给 DeepSeek。

## 发布前检查清单

- `npm test` 通过。
- `npm run build` 通过。
- 手机打开 HTTPS 地址后首屏可用。
- 手机浏览器可“添加到主屏幕”。
- 断网后已缓存页面能打开，或至少有明确失败状态。
- 完成问诊后刷新页面，规划数据仍在。
- CloudBase 云函数已配置 `DEEPSEEK_API_KEY`。
- “测试 AI 服务”能成功，或能明确显示代理/DeepSeek 错误。
- “继续问 AI”能基于当前规划摘要返回回答。
