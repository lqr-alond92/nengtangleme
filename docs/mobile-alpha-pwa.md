# 手机 Alpha PWA 体验说明

本阶段目标是让体验用户直接在自己的手机上使用「能躺了吗」，不登录、不注册、不做云端数据保存。应用只需要一个 HTTPS 静态地址来分发前端文件；用户的规划数据和模型 API Key 都保存在用户当前手机浏览器里。

## 为什么不是 `127.0.0.1`

`127.0.0.1` 永远指向“当前设备自己”：

- 在开发者电脑上打开 `http://127.0.0.1:5173`，访问的是开发者电脑。
- 在用户手机上打开 `http://127.0.0.1:5173`，访问的是用户手机自己。

手机体验用户不能直接访问开发者电脑上的 localhost。真正可用的手机 Alpha 需要一个手机能打开的 HTTPS PWA 地址。

## 推荐交付形态

```text
开发者构建 dist/
        |
        v
HTTPS 静态托管地址
        |
        v
体验用户手机浏览器打开并添加到主屏幕
        |
        v
规划数据 + API Key 保存在手机浏览器 localStorage
```

静态托管只负责分发 HTML、CSS、JS、图标和 manifest，不保存用户的计划数据，不接收用户的模型 API Key。

## 开发者部署步骤

1. 构建前端：

```bash
npm install
npm run build
```

2. 将 `dist/` 目录部署到 HTTPS 静态托管服务。

可选托管方式：

- GitHub Pages
- Cloudflare Pages
- Netlify
- Vercel
- 对象存储静态网站
- 任意能提供 HTTPS 的静态文件服务

3. 把部署后的 HTTPS 地址发给体验用户。

如果使用国内自有域名、国内云厂商或正式生产域名，按对应平台和地区要求处理备案、域名解析和 HTTPS。备案审批期间，可以先使用托管平台提供的临时 HTTPS 域名做小范围体验。

## 体验用户操作

1. 用手机浏览器打开体验地址。
2. 在浏览器菜单中选择“添加到主屏幕”。
3. 从手机桌面打开「能躺了吗」。
4. 完成首次问诊。
5. 进入“模型”页，选择服务商并填写自己的 API Key。
6. 点击“测试模型调用”。
7. 回到“规划”页继续问 AI。

## 数据与 Key 存储边界

当前实现使用浏览器 `localStorage`：

- `neng_tang_plan_v1`：用户规划数据。
- `neng_tang_onboarding_complete_v1`：是否完成首次问诊。
- `neng_tang_model_settings_v1`：模型服务商、模型名、接口地址和 API Key。

这些数据只存在当前浏览器环境里。用户换手机、换浏览器、清理站点数据或卸载 PWA 后，数据可能丢失。本阶段不提供账号同步和云端恢复。

## 模型调用边界

当前模型调用是纯前端直连：

```text
用户手机浏览器 -> 模型服务商 API
```

这能满足“API Key 不上传到能躺了吗服务器”的约束，但有一个现实限制：模型服务商必须允许浏览器跨域调用。如果服务商接口不允许 CORS，用户在“测试模型调用”时会看到网络错误或 CORS 拦截。

如果遇到 CORS 拦截，有三种后续选择：

1. 换一个允许浏览器直连的模型服务商或接口。
2. 让体验用户填写自己的代理地址，由用户自己控制 Key 的流转。
3. 后续改为服务端代理。这个方案体验更稳定，但 Key 会经过服务器，和当前“Key 不上传”的边界不同，需要重新设计隐私和安全说明。

## 发布前检查清单

- `npm test` 通过。
- `npm run build` 通过。
- 手机打开 HTTPS 地址后首屏可用。
- 手机浏览器可“添加到主屏幕”。
- 断网后已缓存页面能打开，或至少有明确失败状态。
- 完成问诊后刷新页面，规划数据仍在。
- 填写 API Key 后刷新页面，模型配置仍在。
- “测试模型调用”能成功，或能明确显示 CORS/网络错误。
- “继续问 AI”能基于当前规划摘要返回回答。
