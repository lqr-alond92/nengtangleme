## 1. 这份文档解决什么

你现在已经买好了域名，也已经有 OSS Bucket。接下来最容易混乱的是：

- 域名应该解析到 OSS，还是 ECS？
- 前端和后端要不要分两个域名？
- API 怎么走 HTTPS？
- ICP 备案、公安备案和域名解析是什么关系？
- 现在应该先配置什么，什么可以晚点做？

本文件给出第一版正式 Web App / PWA 的阿里云部署地图。

## 2. 推荐最终架构

```text
用户浏览器
  ↓ HTTPS
nengtangleme.cn
  ↓
前端 Web App / PWA
  ↓ HTTPS API
api.nengtangleme.cn
  ↓
ECS Node.js API
  ↓
PostgreSQL 数据库
Redis
阿里云百炼 / 通义千问
日志与监控
```

第一版建议：

- `nengtangleme.cn`：前端站点；
- `www.nengtangleme.cn`：跳转到主域名；
- `api.nengtangleme.cn`：后端 API；
- 不把 API 和前端混在同一个静态 OSS Bucket 里；
- 不直接把模型 Key 放前端；
- 不做原生 App；
- 不做小程序。

## 3. 域名规划

### 3.1 域名用途表

| 域名 | 用途 | 第一版建议 | 说明 |
| --- | --- | --- | --- |
| `nengtangleme.cn` | 正式 Web App | 保留给前端 | 用户访问主入口 |
| `www.nengtangleme.cn` | WWW 别名 | 跳转到主域名 | 方便用户习惯输入 |
| `api.nengtangleme.cn` | 后端 API | 指向 ECS / API 网关 | 登录、保存、AI、计算都走这里 |
| `staging.nengtangleme.cn` | 内测前端 | 可选 | 内测环境使用 |
| `staging-api.nengtangleme.cn` | 内测 API | 可选 | 内测后端使用 |

### 3.2 为什么要分 `api.nengtangleme.cn`

因为你的产品不是纯静态网页。它需要：

- 登录；
- 保存用户计划；
- 数据库；
- AI 对话；
- 通义千问 API Key；
- 日志；
- 限流；
- 合规记录。

这些都必须放后端，不能放 OSS 静态站点。

## 4. 第一版部署方式

### 4.1 推荐：ECS 同时托管前端和后端

第一版最简单、最好理解：

```text
ECS
  ├─ Nginx
  │   ├─ nengtangleme.cn → 前端 dist/
  │   └─ api.nengtangleme.cn → Node.js API
  ├─ Node.js API
  └─ 后端环境变量
```

优点：

- 备案、HTTPS、反向代理逻辑清晰；
- 前端和后端都在一台机器，排查问题容易；
- 不需要一开始就折腾 CDN、OSS 静态站点、跨域等问题；
- 适合个人主体和内测阶段。

缺点：

- 静态资源性能不如 OSS + CDN；
- 后续用户多了需要拆分。

### 4.2 第二阶段：前端 OSS + CDN，后端 ECS

内测稳定后再升级：

```text
nengtangleme.cn
  ↓
CDN
  ↓
OSS 静态站点

api.nengtangleme.cn
  ↓
ECS / SLB / API Gateway
```

你当前已有 OSS Bucket：`neng-tang-le-ma-1158245633`。它可以留作第二阶段前端静态资源或文件存储使用。

### 4.3 暂时不推荐

第一版暂时不要：

- 不要直接把 `nengtangleme.cn` 解析到 OSS 并认为上线完成；
- 不要把 API Key 写进前端；
- 不要上来就买很多云产品；
- 不要同时做 App、小程序、Web；
- 不要把 `api.nengtangleme.cn` 指向 OSS。

## 6. DNS 解析规划

### 6.1 内测第一版

如果使用一台 ECS：

| 主机记录 | 记录类型 | 记录值 | 说明 |
| --- | --- | --- | --- |
| `@` | A | ECS 公网 IP | `nengtangleme.cn` |
| `www` | CNAME 或 A | 主域名或 ECS IP | 跳转到主域名 |
| `api` | A | ECS 公网 IP | `api.nengtangleme.cn` |

说明：

- `@` 表示根域名 `nengtangleme.cn`；
- `api` 表示 `api.nengtangleme.cn`；
- 如果前后端都在同一台 ECS，`@` 和 `api` 可以先指向同一个公网 IP；
- 由 Nginx 根据域名区分前端和后端。

### 6.2 第二阶段 OSS + CDN

| 主机记录 | 记录类型 | 记录值 | 说明 |
| --- | --- | --- | --- |
| `@` | CNAME | CDN 分配域名 | 前端静态站点 |
| `www` | CNAME | `nengtangleme.cn` 或 CDN 域名 | WWW 别名 |
| `api` | A 或 CNAME | ECS / SLB / API 网关 | 后端 API |

## 7. HTTPS 规划

需要至少两张证书或一张通配符证书：

```text
nengtangleme.cn
www.nengtangleme.cn
api.nengtangleme.cn
```

第一版可以申请免费证书：

- 主域名证书：`nengtangleme.cn`，包含 `www.nengtangleme.cn`；
- API 证书：`api.nengtangleme.cn`。

Nginx 负责：

- HTTP 自动跳转 HTTPS；
- `nengtangleme.cn` 返回前端静态文件；
- `api.nengtangleme.cn` 反向代理到 Node.js API；
- 设置安全响应头。

## 8. Nginx 路由规划

### 8.1 前端站点

```text
nengtangleme.cn
  ↓
/var/www/nengtangleme/dist
```

前端是 React SPA，需要所有页面 fallback 到 `index.html`。

### 8.2 后端 API

```text
api.nengtangleme.cn
  ↓
http://127.0.0.1:8787
```

生产环境 Node API 不直接暴露公网端口，只让 Nginx 反向代理。

## 9. 生产环境变量

后端 `.env.production` 需要：

```text
NODE_ENV=production
API_PORT=8787
APP_ORIGIN=https://nengtangleme.cn
DATABASE_URL=postgres://...
REDIS_URL=redis://...
COOKIE_SECRET=...
JWT_SECRET=...
QWEN_API_KEY=...
QWEN_BASE_URL=...
QWEN_MODEL=...
LOG_LEVEL=info
```

前端需要：

```text
VITE_API_BASE_URL=https://api.nengtangleme.cn
```

当前开发期前端通过 Vite proxy 使用 `/api`，生产部署时可以继续让 Nginx 转发，也可以改成显式 API 域名。

## 10. 备案关系

### 10.1 ICP 备案

你是个人主体，网站内容建议写成：

```text
家庭财务规划测算工具
```

避免写成：

```text
投资顾问
理财服务
金融服务
证券投资建议
保险咨询
```

原因：

- 当前产品是测算与规划工具；
- 不能暗示提供持牌金融服务；
- AI 输出也不能给具体投资产品建议。

### 10.2 公安联网备案

ICP备案通过并上线后，再做公安联网备案。

页面底部最终需要：

```text
ICP备案号
公安联网备案号
用户协议
隐私政策
AI 使用说明
联系方式
```

### 10.3 App 备案

当前只做 Web App / PWA，先不做 App 备案。

后续如果上架：

- iOS App；
- Android App；
- 小程序；
- 快应用；

再进入 App 备案流程。

## 11. 推荐实施顺序

### 第 1 步：确认域名实名

在阿里云域名控制台确认：

- `nengtangleme.cn` 已购买成功；
- 实名认证已通过；
- 域名状态正常。

### 第 2 步：购买 ECS

推荐：

```text
地域：华东 2（上海）
规格：2 核 4G
系统：Ubuntu LTS
公网 IP：需要
带宽：按量或 3-5 Mbps 起步
```

选择华东 2 的好处：

- 和你现有 OSS Bucket 区域一致；
- 后续资源管理更简单。

### 第 3 步：部署后端和前端

先把代码部署到 ECS：

```text
/srv/nengtangleme/app
```

构建前端：

```bash
npm install
npm run build
```

启动后端：

```bash
npm run dev:api
```

生产期后端要改成进程守护，例如 systemd 或 pm2。

### 第 4 步：配置 Nginx

配置两个站点：

```text
nengtangleme.cn
api.nengtangleme.cn
```

### 第 5 步：配置 DNS

把：

```text
@   → ECS 公网 IP
api → ECS 公网 IP
www → ECS 公网 IP 或跳转到主域名
```

### 第 6 步：申请 HTTPS

先让两个域名都能 HTTPS 访问：

```text
https://nengtangleme.cn
https://api.nengtangleme.cn/api/health
```

### 第 7 步：开始 ICP 备案

准备：

- 个人身份证；
- 域名；
- 服务器；
- 网站名称；
- 网站说明；
- 联系电话；
- 邮箱；
- 真实性核验。

### 第 8 步：内测

先邀请 20-50 人。

观察：

- 登录是否顺畅；
- 数据是否保存；
- 结果页是否看得懂；
- AI 输出是否安全；
- 模型调用成本；
- 用户最常问什么。
