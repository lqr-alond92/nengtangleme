## 1. 先记住这条主线

如果 `nengtangleme.cn` 要解析到中国内地服务器并正式对外访问，就要先完成 ICP 备案。备案通过前，不要把域名正式解析到可访问的网站首页。

推荐顺序：

1. 域名实名认证。
2. 准备可备案的阿里云中国内地服务器。
3. 提交 ICP 备案。
4. 通过后配置 DNS 解析。
5. 部署 HTTPS。
6. 网站首页底部展示 ICP 备案号。
7. 网站开通后 30 天内提交公安联网备案。

参考阿里云官方说明：

- 中国内地服务器需要在服务器所属接入商完成备案。

- ECS 和轻量应用服务器用于备案时，需要中国内地节点，并满足包月 3 个月及以上等条件。
- 备案成功后再开通网站，并在首页底部添加 ICP 备案号链接。

## 3. ECS、轻量服务器和备案服务码怎么选

### 推荐选择

第一版建议买阿里云 ECS：

- 地域：华东 2（上海）或离用户更近的中国内地地域。
- 规格：2 核 4G 起步。
- 系统：Ubuntu LTS 或 Alibaba Cloud Linux。
- 带宽：需要公网带宽。
- 购买周期：至少 3 个月。

为什么优先 ECS：

- 更适合同时跑 Nginx、Node.js API、后端任务和日志。
- 后续接 PostgreSQL、Redis、SSL、备份更灵活。
- 备案、部署、排查问题都更直观。

轻量应用服务器也可以：

- 适合预算更紧、流量较低的内测期。
- 也要选中国内地节点，并购买至少 3 个月。
- 后续如果需要更复杂的运维能力，可以迁移到 ECS。

暂时不建议只用 OSS：

- OSS 可以放前端静态文件，但当前产品需要登录、保存、AI 调用和后端 API。
- API Key 不能放在前端。
- 第一版用一台 ECS 同时托管前端和后端最简单。

### 备案服务码怎么理解

备案服务码就是把“这次备案的网站”和“阿里云这台服务器”关联起来的一串码。

- 如果备案账号和购买服务器账号是同一个阿里云账号：通常不需要提前手动申请，备案流程里直接选择服务器即可。
- 如果服务器在另一个账号：需要服务器所在账号授权备案服务码给备案账号。
- 不要购买来路不明的备案服务码。

## 4. DNS 解析规划

### 备案前

备案前可以先不正式解析到生产站点。

如果阿里云备案流程要求做临时访问验证，按阿里云页面提示操作即可。不要在备案未通过前把真实产品首页公开开放。

### ICP 备案通过后

如果第一版前后端都部署在同一台 ECS：

| 主机记录 | 类型 | 记录值 | 用途 |
| --- | --- | --- | --- |
| `@` | A | ECS 公网 IP | `nengtangleme.cn` 前端主站 |
| `www` | A 或 CNAME | ECS 公网 IP 或 `nengtangleme.cn` | 访问后 301 跳转到主域名 |
| `api` | A | ECS 公网 IP | `api.nengtangleme.cn` 后端 API |

Nginx 根据访问域名分流：

- `nengtangleme.cn`：返回前端构建产物。
- `www.nengtangleme.cn`：301 跳转到 `https://nengtangleme.cn`。
- `api.nengtangleme.cn`：反向代理到本机 Node.js API，例如 `127.0.0.1:8787`。

### 第二阶段

如果后续前端迁到 OSS + CDN：

| 主机记录 | 类型 | 记录值 | 用途 |
| --- | --- | --- | --- |
| `@` | CNAME | CDN 分配域名 | 前端 |
| `www` | CNAME | 主域名或 CDN 域名 | WWW 别名 |
| `api` | A 或 CNAME | ECS、SLB 或 API 网关 | 后端 API |

### 网站信息

| 字段 | 建议填写 |
| --- | --- |
| 网站域名 | `nengtangleme.cn` |
| 网站名称 | 优先尝试 `家庭目标整理记录` 或 `生活目标整理记录` |

网站名称注意：

- 不建议写“能躺了吗平台”，个人网站名称里“平台”通常是风险词。
- 不建议写“家庭财务规划平台”“资产规划工具”“理财助手”“投资规划”等。
- 不要写“金融”“投资”“证券”“保险”“理财”“收益”“资产管理”“财富管理”“贷款”“基金”“股票”。

如果阿里云或管局要求网站名称与实际页面一致，可以把页面标题和备案名称统一成更稳的口径，例如：

- 备案网站名称：`家庭目标整理记录`
- 页面展示名称仍可保留产品名：`能躺了吗`
- 首页底部或关于页说明：这是一个个人生活目标与预算记录工具，不提供投资建议。

| 字段 | 建议填写 |
| --- | --- |

## 6. HTTPS 规划

ICP 备案通过并完成 DNS 解析后申请证书：

- `nengtangleme.cn`
- `www.nengtangleme.cn`
- `api.nengtangleme.cn`

第一版可以用免费单域名证书，也可以用一张支持多个域名的证书。部署后要求：

- HTTP 自动跳转 HTTPS。
- `www` 自动跳转主域名。
- API 只通过 HTTPS 暴露。
- Node.js API 端口不直接暴露公网，只让 Nginx 反向代理。

## 7. 公安联网备案

ICP 备案通过并正式开站后 30 天内，登录全国互联网安全管理服务平台提交公安联网备案。

公安备案一般会用到：

- ICP 备案号。
- 域名：`nengtangleme.cn`。
- 网站名称。
- 网站负责人信息。
- 接入服务商：阿里云。
- 服务器 IP。
- 服务类型和网站简介。

公安备案通过后，网站首页底部通常需要展示公安备案号，并链接到公安备案平台。

## 9. 上线前合规文案提醒

产品页面和 AI 输出里要保持同一套口径：

- 可以说：生活目标整理、预算记录、家庭收支测算、长期生活安排参考。
- 不要说：投资建议、理财方案、收益预测、财富管理、资产配置服务、证券基金推荐。
- 必须说：结果仅供参考，不构成金融、投资、证券、保险或税务建议。

建议正式上线前补齐：

- 用户协议。
- 隐私政策。
- 个人信息收集清单。
- 非投资建议免责声明。
- AI 使用说明。
- 模型服务公示。
- 账号注销说明。
