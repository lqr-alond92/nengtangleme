# AI 结果页重构方案

本方案记录当前体验版本暴露的问题，以及下一版结果页的重构方向。目标是让结果页从“规则展示页”变成“AI 正在帮用户理解长期处境”的体验。

## 当前问题

1. 输入流程仍是旧对话，和 MVP 家庭财务画像不匹配。
2. 新问题出现后没有自动滚动到当前问题，用户需要手动找位置。
3. Loading 直接显示“正在生成结构化分析 DeepSeek”，过于技术化。
4. 结果页结构混乱，核心结论和资产重构的关系不清楚。
5. 风险、后续步骤、具体行动点呈现过重，不够清晰。
6. 行动点缺少“设置目标 / 更改目标 / 加入计划”的操作入口。

## 结果页目标

结果页要让用户逐步感受到：

1. 系统正在理解他的输入。
2. 系统正在重构他的真实资产。
3. 系统给出一个有洞见的核心结论。
4. 系统指出风险。
5. 系统给出可以执行的行动。

不要一次性把所有内容全部铺开。

## 输出节奏

采用类打字机的渐进式输出。

### 交互要求

- 结果页进入后，先出现 AI 头像和简短思考态。
- 每个区块按顺序出现，不一次性全部展示。
- 每段核心文字使用打字机效果逐字出现。
- 区块之间保留短暂停顿，让用户能跟上理解节奏。
- 新区块出现时，页面自动滚动到新区块顶部。
- 用户可以继续手动滚动，不强行锁定页面。

### 推荐输出顺序

1. AI 思考中
2. 重构后的真实资产
3. 核心结论
4. 风险
5. 后续步骤
6. 具体行动点
7. 动态追问

## Loading 设计

不要展示模型名、接口名或“结构化分析”等技术词。

建议文案：

- 正在重新整理你的真实资产
- 正在把未来收入和目标折算到今天
- 正在判断这套长期规划是否有风险

视觉形式：

- AI 头像轻微呼吸动效。
- 3 个点的思考动画。
- 进度不显示百分比。
- 不出现 `DeepSeek / deepseek-chat`。

## 重构后的真实资产

这一部分只回答一个问题：

**用户真正能用于支撑未来目标的钱是多少？**

不要在这里展示目标值，避免用户在第一张卡里同时看到“资产”和“目标”，造成理解混乱。

### 展示内容

- 当前灵活资产
- 未来结余折现值
- 重构后可用资产现值

### 公式

```text
当前灵活资产 + 未来结余折现值 = 重构后可用资产
```

固定资产默认不计入，除非用户明确愿意出售、抵押或出租。

## 核心结论

核心结论不能只是加减法，也不能只说覆盖率。

它必须回答：

1. 用户当前遇到的问题是什么？
2. 当前赚钱能力是否能支撑未来目标？
3. 长期规划的最大卡点在哪里？
4. 用户应该如何理解自己的处境？

### 判断逻辑

不要同时展示“减必要、减想要、减全部”三张相似卡片。

优先使用合并逻辑：

1. 先判断：重构后可用资产是否能覆盖全部目标现值。
2. 如果全部目标能覆盖：
   - 给出对号。
   - 说明仍需关注的风险，比如收入持续性、灵活资产过低、目标金额波动。
3. 如果全部目标不能覆盖：
   - 再拆分必要目标和想要目标。
   - 判断必要目标是否能覆盖。
   - 判断想要目标是否需要延期、降配或排序。

### 单卡展示方式

核心结论尽量整合在一张卡里：

```text
重构后可用资产：1056万
全部目标现值：1022万
结果：有 34 万余量，但余量很薄
```

如果全部目标不能覆盖，则在同一张卡内展开：

```text
全部目标：不足
必要目标：可覆盖
想要目标：需要排序或延期
```

### 图形表达

- 对号：可以覆盖。
- 叉号：无法覆盖。
- 感叹号：勉强覆盖，但安全余量薄。

图形只服务于判断，不要做成三张重复图。

## 洞见表达

核心结论必须有“洞见”，不能只写：

```text
1056 万 - 1022 万 = 34 万
```

它应该进一步翻译成用户能感同身受的话：

```text
你的问题不是账面资产少，而是可动用资产太薄，未来目标又高度依赖接下来 20 年持续赚钱。一旦收入中断、教育支出提前或换房预算上升，当前看似有余量的方案会迅速变紧。
```

Prompt 必须要求模型输出：

- 一句数字判断；
- 一句处境翻译；
- 一句最大卡点；
- 一句下一步方向。

## 风险

风险区块要简洁，不做长表格。

每个风险卡只保留：

- 风险标题
- 一句话原因
- 会影响什么
- 对应动作

最多展示 3 个风险。

## 后续步骤

后续步骤最多 3 条。

要求：

- 按优先级排序。
- 每一步都能对应一个用户可操作动作。
- 不要出现泛泛的“提高收入”“减少支出”。

## 具体行动点

行动点改为左右滑动卡片。

每张卡片包含：

- 原因：为什么现在要做这件事。
- 方案：具体怎么做。
- 按钮：设置 / 更改 / 加入计划。

### 操作入口

行动点需要能进入对应页面：

- 设置目标：进入目标页并预填目标。
- 更改目标：进入目标编辑状态。
- 调整资产：进入资产页。
- 加入计划：生成计划清单。

### 计划清单

用户完成设置或更改后，系统应生成对应计划清单。

示例：

- 本月补足灵活资产到 25 万。
- 将换房目标从 8 年后调整到 10 年后。
- 教育金目标单独建账，每月投入 1.25 万。

## 动态追问

继续追问由大模型根据用户实际画像生成。

要求：

- 不写死固定问题。
- 问题应围绕当前最大不确定性。
- 每次最多展示 3 个。
- 问题要短，适合手机横向滚动。

## 下一版实现优先级

1. 先改输入流程，让输入和 MVP 家庭财务画像一致。
2. 再改结果页输出节奏，加入打字机和自动滚动。
3. 重构“真实资产”和“核心结论”两张核心卡。
4. 简化风险和后续步骤。
5. 行动点改为可滑动卡片，并接入设置目标/更改目标/计划清单。
