# PRD-002 AI 结果页与 DeepSeek 结构化输出

状态：Draft  
日期：2026-06-11  
适用版本：Local Alpha / CloudBase Demo  
原型依据：`docs/04-原型与工作流/result-page-ai-rework-demo.html`

## 1. 背景

当前产品已经完成静态站点部署、CloudBase 云函数代理和 DeepSeek 初步接入。用户可以在手机浏览器中进入「能躺了吗」，填写家庭资产、负债、收入、支出和未来目标，并得到 AI 分析。

但上一版结果页存在三个问题：

1. 结果页信息仍像固定规则报告，AI 输出的“千人千面”感不足。
2. 资产重构、核心结论、风险、行动建议之间的层级不清楚。
3. DeepSeek 输出缺少稳定格式，前端无法可靠渲染结构化视觉模块。

本 PRD 定义下一版 AI 结果页：用 DeepSeek 生成结构化判断，前端按照固定信息架构和视觉节奏逐步展示。

## 2. 产品目标

让用户在 2 分钟内完成一次家庭财务长期规划判断，并能看懂三件事：

1. 现在的钱和未来收入合起来，按今天看大概值多少钱。
2. 扣掉已有负债后，真正能用于未来目标的钱是多少。
3. 当前目标是否站得住，风险在哪里，下一步该做什么。

## 3. 核心概念

### 3.1 用户可理解术语

不要在页面上直接使用“净现值”“折现法”等专业词。

统一使用：

```text
未来收入的今天价值
```

说明文案：

```text
把未来每年能留下的钱，换算成今天看值多少钱，再和现在手里的钱放在一起判断。
```

### 3.2 资产重构口径

结果页不直接展示复杂公式，而展示两个核心结果：

1. 合起来能规划的钱  
   由“现在手里的钱 + 未来收入的今天价值”组成。

2. 扣掉已有负债后，真正能用于目标的钱  
   由“合起来能规划的钱 - 当前总负债”得到。

示例：

```text
合起来能规划的钱：1,056 万
= 现在手里的钱 15 万 + 未来收入的今天价值 1,041 万

扣掉已有负债后，真正能用于目标的钱：606 万
= 1,056 万 - 当前总负债 450 万
```

`606 万` 是后续判断目标是否够用的核心口径。

## 4. 页面信息架构

结果页按以下顺序展示：

1. 用户问题气泡
2. AI 解释气泡
3. Loading 思考态
4. 重构后的真实资产
5. 核心结论
6. 风险
7. 行动建议
8. 动态追问

## 5. 输出节奏

每个根模块必须遵循：

1. 先显示模块标题。
2. 文案用打字机效果逐字出现。
3. 文案结束后，再展示该模块的结构化卡片或可视化内容。

不能出现“文字还没打完，1056 / 606 或风险卡已经提前出现”的情况。

## 6. 模块需求

### 6.1 Loading 思考态

禁止显示：

- DeepSeek
- deepseek-chat
- 正在生成结构化分析
- 接口名、模型名、技术过程

推荐展示：

- 正在整理你的家庭财务地图
- 重构真实资产
- 换算未来收入的今天价值
- 判断哪些风险会挤压计划

### 6.2 重构后的真实资产

模块目标：让用户明白“后续判断用哪个钱数”。

展示两张同级结果卡：

1. 合起来能规划的钱
   - 主数字：`planningPoolPv`
   - 说明：现在手里的钱 + 未来收入的今天价值

2. 扣掉已有负债后，真正能用于目标的钱
   - 主数字：`debtAdjustedAvailablePv`
   - 说明：已扣除当前总负债

视觉要求：

- 两张卡同等重要，不做一主一辅。
- 卡片高度要克制，不占满一整屏。
- 债务不要作为中间大卡，而是作为第二张卡的说明。

### 6.3 核心结论

模块目标：不是展示算式，而是翻译用户处境。

必须回答：

1. 当前赚钱能力能不能支撑目标？
2. 全部目标是否能覆盖？
3. 如果全部目标不能覆盖，必要目标能否守住？
4. 想要目标还差多少钱或需要如何排序？

展示结构：

- 一句话 headline
- 一段处境洞见
- 数字拆解：
  - 债务后可规划资产
  - 全部目标按今天算
  - 全部目标缺口或余量
- 必要目标判断
- 想要目标判断

### 6.4 风险

最多展示 3 个风险。

每个风险包含：

- 标题
- 原因
- 影响
- 对应动作

风险必须来自用户数据，不输出泛泛理财常识。

### 6.5 行动建议

行动建议采用横向滑动卡片。

每张卡固定为三层：

1. 标题
2. 原因
3. 目标
4. 操作按钮

不要再把“月度、周度、预计时间”作为散落标签展示；这些信息应合并进“目标”描述里。

行动点类型至少支持：

1. 设置计划
2. 更改目标
3. 建立账户
4. 复核现金流

点击按钮后打开计划弹窗，弹窗包含：

- 当前状态
- 目标值
- 月度执行方案
- 周度执行方案
- 预计完成时间
- 生成后的计划清单

### 6.6 动态追问

追问问题由 DeepSeek 根据用户当前画像生成，不写死固定问题。

要求：

- 最多 3 个。
- 每个问题短，适合手机横向滚动。
- 围绕当前最大不确定性，例如收入下降、换房延期、应急金、必要目标和想要目标排序。

DeepSeek 必须返回合法 JSON，不要 Markdown，不要解释 JSON。

### 7.2 字段约束

- 所有金额字段优先返回数字，单位为元。
- 页面文案字段要使用中文短句。
- `actions.actionPoints` 最多 4 个。
- `risks` 最多 3 个。
- `followUpQuestions` 最多 3 个。
- 不要输出投资品推荐、具体证券买卖建议或收益承诺。

## 8. 验收标准

1. 结果页 Loading 不暴露模型名和技术流程。
2. 每个模块先打字，后出现结构化卡片。
3. 资产模块只突出两个数字：合起来能规划的钱、扣债后真正能用于目标的钱。
4. 核心结论有洞见，不只是公式。
5. 行动建议按“原因 / 目标 / 操作”分层。
6. 操作按钮能打开计划弹窗。
7. DeepSeek 返回结构化 JSON 后，前端能稳定展示；字段缺失时有兜底。
8. 继续追问来自模型输出，最多展示 3 个。
